# AI-driven 城市智慧停车管理与诱导系统

本仓库用于保存软件工程实训(A)第19组的项目文档、教师任务截图、交付物模板和自动化生成脚本。项目主题为 **AI-driven 城市智慧停车管理与诱导系统**，实训阶段重点完成项目立项、项目计划、需求分析、系统设计和日常报告等文档交付。

## 项目概述

系统面向城市停车资源分布不均、车位信息割裂、停车场运营效率不足、监管统计滞后等问题，设计覆盖车主端、停车场运营端和城市监管端的智慧停车管理与诱导方案。

需求与设计严格围绕 18 个一级业务模块展开：

| 优先级 | 模块 |
| --- | --- |
| P0 核心闭环 | 停车场数据联网接入、城市停车一张图、智能导航与车位预约、车牌识别与无感支付、车场运营管理后台、车主移动端 APP/小程序 |
| P1 重要能力 | 路内停车位视觉检测、停车诱导信息发布、停车行为大数据分析、动态定价策略引擎、共享停车管理平台、反向寻车导航、城市停车监管平台 |
| P2 扩展能力 | 充电车位智能管理、违停自动抓拍与取证、长期停车月卡管理、设备运维管理平台、商圈停车联合营销 |

实训环境不强依赖真实 AI 模型、GIS 地图、支付通道或硬件设备，上述能力在文档中通过模拟数据、预留接口、状态面板和可替换服务进行设计。

## 文档生成脚本

脚本位于 `tools/`，主要用于从模板和 Markdown 源生成或更新 Word 文档。

| 脚本 | 用途 |
| --- | --- |
| `build_project_start_report.py` | 生成项目立项 DOCX |
| `build_srs_v2.py` | 基于需求模板生成需求分析 DOCX，并生成用例图 |
| `build_hld_from_template.py` | 基于 HLD 模板母版生成系统设计 DOCX |
| `expand_week3_deliverables.py` | 在不改变模板结构的前提下扩充需求分析和系统设计细节 |
| `update_docx_deliverables.py` | 批量更新已有 DOCX 的部分表格和说明 |

推荐使用 Windows + Python 环境运行。涉及 `.doc` 模板转换时，本地 Microsoft Word COM 可用于把老式 Word 模板转换为 `_template_base.docx`。

示例：

```powershell
python tools/build_srs_v2.py
python tools/build_hld_from_template.py
python tools/expand_week3_deliverables.py
```

## 模板与格式约束

所有正式 DOCX 产物应遵循 `交付物/巩固式解密模板/` 中对应模板的结构和格式。生成文档时应采用“模板文件 + 内容替换”的方式，而不是仅模仿样式。

已经处理的关键约束：

- 不修改 `交付物/` 模板目录。
- 需求分析基于 SRS 模板母版生成。
- 系统设计基于 HLD 模板母版生成。
- 文档内容按教师任务发布时间推进，避免需求、设计前后不连贯。
- 03 和 04 两份核心 DOCX 已扩充细节，但不新增模板外章节。
- 04 系统设计已补充 2026-06-05 要求的第5章出错处理设计和设计评审结论。

## 备注

`_render_check` 下的 PDF 是本地 Word 导出的检查文件，用于辅助确认 DOCX 可打开、可导出。由于不同机器的 Office、LibreOffice 和字体环境可能不同，提交前建议在本机 Word 中再打开最终 DOCX 检查页眉、分页、表格和图片显示。
