## 1. 产品概述

### 1.1 产品定位

**智慧停车平台** 是一款面向城市停车场景的 Web 管理系统，核心解决 "车主找车位难、车场管理乱、数据不互通" 三大痛点。产品以车主端（车位搜索→预约→支付→寻车）和管理端（运营后台→数据看板）两条主线，提供完整的停车服务闭环。

### 1.2 产品愿景

> 让每一位车主 "出门就知道停哪儿"，让每一个车场 "数据驱动运营"。

### 1.3 目标用户

| 用户角色 | 核心诉求 | 使用频率 |
|----------|----------|:--------:|
| 私家车主 | 快速查车位、在线预约、无感支付、反向寻车 | 每日 |
| 停车场管理员 | 管理车位状态、查看收入统计、处理异常订单 | 每日 |
| 车位主（共享） | 发布闲置车位时段、获取额外收入 | 按需 |
| 运营主管 | 查看全局运营数据、辅助管理决策 | 每周 |

### 1.4 产品范围

**本期范围（V1.0）**: 车主端 M1-M7 + 管理端 M8-M9 = 9个核心模块

**下期规划（V2.0/V3.0）**: 共享车位 M10 + 违停管理 M11

### 2.2 模块需求明细

#### M1 — 用户注册与登录

| 项 | 说明 |
|----|------|
| **功能描述** | 用户通过手机号+密码注册账号，登录后使用完整功能 |
| **优先级** | P0（核心） |
| **输入** | 手机号、密码、确认密码 |
| **输出** | 注册成功→自动登录；登录成功→跳转车主首页 |
| **业务规则** | ①密码长度≥6位，需包含数字和字母 ②手机号唯一性校验 ③密码 bcrypt 哈希存储 |
| **异常处理** | 手机号已注册→提示"该手机号已注册"；登录失败→提示"用户名或密码错误" |

#### M2 — 停车场信息管理

| 项 | 说明 |
|----|------|
| **功能描述** | 管理员录入/编辑车场信息；车主浏览车场列表与详情 |
| **优先级** | P0（核心） |
| **输入** | 车场名称、地址、经纬度、总车位数、费率（元/小时） |
| **输出** | 车场列表（分页）、车场详情（含实时空余车位数） |
| **业务规则** | ①管理员方可新增/编辑/删除车场 ②车场删除采用软删除（status标记），不物理删除数据 |
| **异常处理** | 数据校验失败→返回具体字段错误提示 |

#### M3 — 车位状态实时查询

| 项 | 说明 |
|----|------|
| **功能描述** | 车主按目的地搜索附近车场，查看各车场实时空余车位 |
| **优先级** | P0（核心） |
| **输入** | 搜索关键词（目的地/地址）、筛选条件（距离/价格/空位数） |
| **输出** | 车场列表（含名称、地址、距离、空余车位、费率） |
| **业务规则** | ①默认按距离排序 ②支持按价格/空位数排序 ③空余车位=总车位-已占用-已锁定 |
| **异常处理** | 无匹配结果→提示"未找到相关停车场" |

#### M4 — 车位在线预约 ⭐ [V0.2修订]

| 项 | 说明 |
|----|------|
| **功能描述** | 车主选定车位后预约锁定15分钟，到期自动释放并通知 |
| **优先级** | P0（核心） |
| **输入** | 用户ID、车位ID、车场ID |
| **输出** | 预约成功→显示倒计时（15分钟）→到场确认/取消预约 |
| **业务规则** | ①车位状态机：free → locked → occupied → free ②锁定15分钟超时自动释放 ③**同一用户在任何时刻只能持有1个active预约（前后端双重校验）** 【修订#2】 ④超时后前端倒计时归零弹窗"预约已过期"，预约状态→expired 【修订#3】 ⑤后端被动清理：每次查询预约时检查expire_at，超时自动释放 |
| **异常处理** | 车位已被锁定→提示"该车位已被预约"；用户已有活跃预约→前端按钮置灰+后端拒绝创建【修订#2】 |
| **并发控制** | 预约创建使用数据库行级锁：`UPDATE spot SET status='locked' WHERE id=? AND status='free'`【QA建议】 |

#### M5 — 停车计费与支付 ⭐ [V0.2修订]

| 项 | 说明 |
|----|------|
| **功能描述** | 记录入场/出场时间，按费率计算费用，模拟支付完成结算 |
| **优先级** | P0（核心） |
| **输入** | 入场时间（系统自动记录）、出场时间、费率 |
| **输出** | 费用明细（时长、单价、总价）→确认支付→生成订单 |
| **业务规则** | ①**前15分钟免费（≤15分钟不计费）**【修订#1】 ②**超过15分钟后按1小时起计，之后不足1小时按1小时计**【修订#1】 ③示例：停车1h5min = 计2小时；停车14min = 免费 ④模拟支付：前端展示支付动画2秒→后端变更状态 ⑤订单号唯一索引防重复提交【QA建议】⑥订单状态：parking → completed |
| **异常处理** | 支付确认失败→保留订单parking状态，可重新发起；重复支付→返回"已支付"【QA建议】 |

#### M6 — 停车记录与反向寻车 ⭐ [V0.2修订]

| 项 | 说明 |
|----|------|
| **功能描述** | 车主查看历史停车记录；输入车牌号查看当前停车位置和步行指引 |
| **优先级** | P1（重要） |
| **输入** | 车牌号（查当前）、筛选条件（查历史） |
| **输出** | 历史记录列表；当前车位编号+区域+**纯文字步行指引**【修订#4】 |
| **业务规则** | ①仅展示本人停车记录 ②反向寻车：输入车牌号→匹配当前占用车位→显示"B2层-西区-128号车位，请从西区电梯口直行50米左转"格式的文字指引【修订#4】 ③V1.0不做交互式地图/AR导航 |
| **异常处理** | 未找到→提示"未找到该车辆当前停车记录" |

## 4. 数据需求 ⭐ [V0.2修订]

### 4.1 核心数据表（6张）

> **DDL完整定义将于第1周（6月7日前）由DBA程晓洋输出**【修订#7】

```sql
-- 1. 用户表
users (id INT PK AUTO_INCREMENT, username VARCHAR(50) NOT NULL, 
       phone VARCHAR(20) UNIQUE NOT NULL, password_hash VARCHAR(255) NOT NULL, 
       role VARCHAR(20) DEFAULT 'user', created_at DATETIME DEFAULT CURRENT_TIMESTAMP)

-- 2. 停车场表
parking_lots (id INT PK, name VARCHAR(100) NOT NULL, address TEXT, 
              latitude DOUBLE, longitude DOUBLE, total_spots INT DEFAULT 0, 
              rate_per_hour DECIMAL(10,2) DEFAULT 5.00, 
              admin_id INT, status VARCHAR(20) DEFAULT 'active') -- 软删除支持

-- 3. 车位表
parking_spots (id INT PK, lot_id INT FK→parking_lots, spot_number VARCHAR(20) NOT NULL, 
               floor VARCHAR(10), zone VARCHAR(20), 
               status VARCHAR(20) DEFAULT 'free',  -- free|locked|occupied|disabled
               updated_at DATETIME)

-- 4. 预约记录表
reservations (id INT PK, user_id INT FK→users, spot_id INT FK→parking_spots, 
              lot_id INT FK→parking_lots, 
              status VARCHAR(20) DEFAULT 'active', -- active|confirmed|cancelled|expired
              created_at DATETIME, expire_at DATETIME, confirm_at DATETIME)

-- 5. 停车订单表
parking_orders (id INT PK, order_no VARCHAR(50) UNIQUE NOT NULL, -- 唯一索引防重复
                user_id INT FK→users, spot_id INT FK→parking_spots, 
                lot_id INT FK→parking_lots, 
                entry_time DATETIME, exit_time DATETIME, 
                duration_minutes INT, amount DECIMAL(10,2), 
                status VARCHAR(20) DEFAULT 'parking') -- parking|completed|cancelled

-- 6. 违停记录表
violations (id INT PK, lot_id INT, spot_id INT, plate_number VARCHAR(20), 
            description TEXT, created_at DATETIME, status VARCHAR(20) DEFAULT 'pending')
```

### 4.2 车位状态机

```
         ┌─────────┐
         │  free   │ ← 初始状态
         └────┬────┘
              │ 预约 → lock 15分钟后超时自动回到free
         ┌────▼────┐
         │ locked  │
         └────┬────┘
          ╱       ╲
   到场确认    超时(expire_at)/取消
    ╱             ╲
┌──▼──────┐   ┌───▼───┐
│occupied │   │  free  │
└────┬────┘   └───────┘
     │ 出场结算
┌────▼────┐
│  free   │
└─────────┘
```

## 5. QA质量风险与规避 ⭐ [V0.2新增]

| 风险项 | 说明 | 规避措施 |
|--------|------|----------|
| 状态一致性 | 预约超时由前端倒计时控制，用户关闭页面后无法触发释放 | 后端被动检查：查询预约时判断expire_at，超时自动标记expired+释放车位 |
| 并发预约 | 两个用户同时预约同一车位 | `UPDATE spot SET status='locked' WHERE id=? AND status='free'` 行级锁 |
| 支付幂等性 | 模拟支付重复点击 | 订单号UNIQUE索引，重复请求返回"已支付" |
| 数据完整性 | 车场删除后的关联数据处理 | 软删除（status标记），保留历史数据可追溯 |
