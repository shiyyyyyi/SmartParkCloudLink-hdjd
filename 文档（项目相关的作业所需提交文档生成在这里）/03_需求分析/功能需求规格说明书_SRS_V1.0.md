## 第一章 引言

### 1.1 编写目的

本文档旨在详细描述"AI-driven城市智慧停车管理与诱导系统"的全部功能需求与非功能性需求，作为后续系统设计、编码实现、测试验收的依据。本文档面向：

- **开发团队**：作为编码实现的直接输入
- **测试人员**：作为测试用例编写的依据
- **课程教师**：作为项目评分参考

### 1.2 项目背景

全国停车位缺口超8000万个，城市核心区车位利用率不足60%，车主找车位平均耗时15-30分钟。本项目基于AI视觉感知、大数据分析和物联网技术，构建覆盖路内停车、路外停车场、共享停车等多种场景的城市智慧停车管理与诱导平台。

## 第二章 总体描述

### 2.1 产品视角

本系统是一个B/S架构的Web应用平台，包含**车主端、管理端、数据端**三大子系统，服务于私家车主、停车场管理员、车位主（共享）、城市交管部门四类用户。

```
┌─────────────────────────────────────────────────────────┐
│                  智慧停车管理与诱导平台                    │
├──────────────┬──────────────────┬───────────────────────┤
│   车主端      │     管理端        │      数据端            │
│  (M1-M7)     │   (M2/M8/M9/M13) │   (M3/M7/M9/M17)     │
├──────────────┼──────────────────┼───────────────────────┤
│ ·注册登录     │ ·车场信息管理      │ ·车位状态实时同步       │
│ ·车场搜索     │ ·车位状态管理      │ ·停车行为大数据分析     │
│ ·车位查询预约 │ ·运营收入统计      │ ·城市停车态势监控       │
│ ·计费支付     │ ·设备运维管理      │ ·动态定价引擎          │
│ ·停车记录寻车 │ ·数据可视化看板    │ ·诱导信息发布          │
│ ·共享车位     │                  │                       │
└──────────────┴──────────────────┴───────────────────────┘
```

### 2.2 用户角色

| 角色 | 标识 | 描述 | 关联模块 |
|------|:---:|------|------|
| 私家车主 | USER | 使用停车服务的主要用户，需要找车位、预约、支付、寻车 | M1/M3/M4/M5/M6/M7/M14 |
| 停车场管理员 | ADMIN | 管理单个或多个停车场的运营人员 | M2/M8/M13/M16 |
| 车位主（共享） | OWNER | 将闲置车位出租的个人或单位 | M9/M10 |
| 城市交管部门 | GOV | 监管全市停车态势的管理部门 | M3/M17/M7/M11 |

### 2.3 运行环境

| 项目 | 要求 |
|------|------|
| 浏览器 | Chrome 90+ / Edge 90+ / Firefox 88+ |
| 屏幕分辨率 | ≥1366×768（管理端）/ ≥375×667（移动端响应式） |
| 后端 | Python 3.10+ / FastAPI |
| 数据库 | SQLite 3（本地文件数据库） |
| 部署方式 | 本地开发运行，无需服务器 |

### 2.4 假设与约束

- **假设**：停车场数据为模拟数据（50+车场、500+车位）；支付为模拟流程；AI识别为模拟接口
- **约束**：不接入真实支付API；不接入真实GIS地图；不接入真实车牌识别；数据库使用SQLite本地文件
- **设计约束**：前端Vue3 + Element Plus；后端FastAPI + SQLAlchemy；前后端RESTful JSON通信

### 3.1 用户注册与登录（M1）

| 属性 | 内容 |
|------|------|
| **功能ID** | M1 |
| **功能名称** | 用户注册与登录 |
| **优先级** | P0（核心） |
| **负责人** | 程晓洋 |
| **功能描述** | 提供用户注册、登录、个人信息管理功能，是整个系统的入口。支持三种角色注册：私家车主、车位主、停车场管理员。 |

#### 3.1.1 功能点

| 编号 | 功能点 | 描述 | 输入 | 输出 |
|:---:|------|------|------|------|
| M1-F1 | 用户注册 | 新用户填写注册信息创建账号 | 用户名、密码、手机号、角色选择 | 注册成功/失败提示 |
| M1-F2 | 用户登录 | 已注册用户凭用户名密码登录 | 用户名、密码 | 登录成功→跳转首页；失败→错误提示 |
| M1-F3 | 个人信息查看 | 查看当前登录用户信息 | 无（登录态） | 用户名、手机号、角色、注册时间 |
| M1-F4 | 个人信息修改 | 修改手机号等个人信息 | 新手机号等 | 修改成功/失败提示 |
| M1-F5 | 密码修改 | 修改登录密码 | 旧密码、新密码、确认新密码 | 修改成功/失败提示 |
| M1-F6 | 退出登录 | 清除登录状态 | 无 | 跳转至登录页 |

#### 3.1.2 业务规则

| 编号 | 规则 |
|:---:|------|
| BR-M1-01 | 用户名长度3-20字符，支持中英文数字下划线 |
| BR-M1-02 | 密码长度6-20字符，需包含字母和数字 |
| BR-M1-03 | 手机号需符合中国大陆11位手机号格式 |
| BR-M1-04 | 用户名全局唯一，注册时校验重复 |
| BR-M1-05 | 密码使用bcrypt哈希存储，不可明文 |
| BR-M1-06 | 登录态使用Session管理，有效期24小时 |

#### 3.1.3 异常处理

| 编号 | 异常场景 | 处理方式 |
|:---:|------|------|
| EX-M1-01 | 用户名已存在 | 提示"用户名已被注册"，建议更换 |
| EX-M1-02 | 密码错误 | 提示"用户名或密码错误"，允许重试3次 |
| EX-M1-03 | 登录超时 | 自动跳转登录页，提示"登录已过期，请重新登录" |

#### 3.1.4 验收标准

- [ ] 注册功能：输入合法信息可成功注册，重复用户名被拒绝
- [ ] 登录功能：正确账号密码可登录，错误密码提示且3次后锁定30秒
- [ ] 密码修改：旧密码正确时修改成功，旧密码错误时提示
- [ ] Session：关闭浏览器后需重新登录

### 3.4 车位在线预约（M4）

| 属性 | 内容 |
|------|------|
| **功能ID** | M4 |
| **功能名称** | 车位在线预约 |
| **优先级** | P0（核心） |
| **负责人** | 程子浩 |
| **功能描述** | 车主选定停车场后，可查看具体车位并在线预约锁定。车位锁定后保留15分钟，超时未到场确认则自动释放。车主到场后确认使用，车位状态变为占用。 |

#### 3.4.1 功能点

| 编号 | 功能点 | 描述 | 输入 | 输出 |
|:---:|------|------|------|------|
| M4-F1 | 车位选择 | 在车场内选择具体空闲车位 | 车位ID | 车位详情（编号、楼层、区域） |
| M4-F2 | 预约确认 | 确认预约并锁定车位 | 车位ID、预计到达时间 | 预约成功+15分钟倒计时 |
| M4-F3 | 预约取消 | 取消已预约但未确认的车位 | 预约ID | 取消成功→释放车位 |
| M4-F4 | 到场确认 | 车主到达车场后确认使用 | 预约ID | 车位状态→占用，生成入场记录 |
| M4-F5 | 超时释放 | 15分钟未确认自动释放车位 | 无（系统自动） | 预约→过期，车位→空闲 |
| M4-F6 | 我的预约 | 查看当前用户的所有预约记录 | 无 | 预约列表（状态：待确认/已确认/已取消/已过期） |

#### 3.4.2 业务规则

| 编号 | 规则 |
|:---:|------|
| BR-M4-01 | 同一用户同时只能有1个活跃预约（待确认状态） |
| BR-M4-02 | 预约锁定时间固定15分钟（可配置） |
| BR-M4-03 | 超时未确认：预约状态→expired，车位→free |
| BR-M4-04 | 到场确认：预约状态→confirmed，车位→occupied，生成停车订单 |
| BR-M4-05 | 预约取消：预约状态→cancelled，车位→free |
| BR-M4-06 | 车位状态机：free ↔ locked → occupied → free |

#### 3.4.3 状态机

```
        预约锁定              到场确认         出场结算
free ──────────→ locked ──────────→ occupied ──────────→ free
  ↑                 │                    │
  │    取消/超时     │                    │
  └─────────────────┘                    │
                                         │
                              (车位释放)  ↓
                                         free
```

#### 3.4.4 验收标准

- [ ] 选择空闲车位可成功预约，车位变为"已锁定"
- [ ] 预约成功后显示15分钟倒计时
- [ ] 倒计时结束后车位自动释放
- [ ] 到场确认后车位变为"占用"，生成停车订单
- [ ] 同一用户不能同时预约两个车位

### 3.5 停车计费与模拟支付（M5）

| 属性 | 内容 |
|------|------|
| **功能ID** | M5 |
| **功能名称** | 停车计费与模拟支付 |
| **优先级** | P0（核心） |
| **负责人** | 程子浩 |
| **功能描述** | 车辆出场时系统自动计算停车时长和费用，车主确认后进行模拟支付。支付成功后释放车位，生成完整停车订单记录。支持多时段阶梯费率。 |

#### 3.5.1 功能点

| 编号 | 功能点 | 描述 | 输入 | 输出 |
|:---:|------|------|------|------|
| M5-F1 | 出场结算 | 车主点击出场，计算费用 | 订单ID | 停车时长、费用明细 |
| M5-F2 | 费用计算 | 按入场时间、出场时间、费率计算 | 入场时间、出场时间、费率 | 停车时长（分钟）、应付金额 |
| M5-F3 | 模拟支付 | 模拟支付确认（无需真实支付） | 订单ID、支付确认 | 支付成功→释放车位 |
| M5-F4 | 订单详情 | 查看单条停车订单完整信息 | 订单ID | 车场、车位、时长、费用、状态 |
| M5-F5 | 支付历史 | 查看用户历史支付记录 | 用户ID、时间范围 | 订单列表（支持分页） |

#### 3.5.2 业务规则

| 编号 | 规则 |
|:---:|------|
| BR-M5-01 | 计费公式：费用 = 停车时长(小时) × 每小时费率 |
| BR-M5-02 | 不足1小时按1小时计费（或按分钟计费，可配置） |
| BR-M5-03 | 支持不同车场不同费率（由M2车场信息维护） |
| BR-M5-04 | 支持免费时长（如首15分钟免费，可配置） |
| BR-M5-05 | 支付为模拟操作，点击"确认支付"即完成 |
| BR-M5-06 | 支付成功后车位立即释放为空闲 |
| BR-M5-07 | 支付成功后订单状态→completed |

#### 3.5.3 验收标准

- [ ] 出场时正确计算停车时长和费用

### B.2 关键细化决策

1. **M12-M22处理策略**：由于课题原始需求包含18个功能模块，但部分模块（如AI视觉检测、GIS地图、真实支付）在实训环境下无法完整实现。经RA分析，采用"需求完整定义+编码简化实现"策略——在SRS中完整定义22个模块的需求规格，编码阶段P0/P1模块完整实现，P2*模块实现CRUD+模拟数据展示版本。

2. **功能点粒度**：每个模块拆分为4-6个功能点，每个功能点定义明确的输入/输出和验收标准，确保可测试。

3. **状态机设计**：对M4预约模块设计了完整的状态机（free→locked→occupied→free），明确各状态转换条件。

4. **模拟数据策略**：所有AI识别、GIS地图、支付功能均采用模拟数据实现，确保系统可在本地完整运行演示。
