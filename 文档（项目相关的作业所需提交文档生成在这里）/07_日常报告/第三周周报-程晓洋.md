## 四、技术方案（本周新增）

### HLD设计要点：
- **系统架构**：前后端分离B/S架构，Vue3 SPA + FastAPI RESTful + SQLite
- **模块设计**：22模块按P0/P1/P2/P2*四级优先级分层设计
- **数据库设计**：8张核心表（含共享车位表、设备表等扩展预留）+ ER图 + 索引设计
- **接口设计**：40+ RESTful API端点，统一返回格式`{code, msg, data}`
- **前端设计**：Vue Router路由表 + 组件树 + Pinia状态管理
- **后端设计**：FastAPI目录结构 + 业务服务层 + 认证中间件

### SRS设计要点：
- 22个功能模块完整需求规格（含M12-M22扩展模块模拟实现策略）
- 每模块含功能点拆分（4-6个F点）、业务规则（BR）、异常处理（EX）、验收标准
- 需求追踪矩阵（RTM）覆盖全部模块
- 8项评审问题全部修订闭合

## 五、遇到的问题与解决

| 问题 | 解决方案 |
|------|------|
| 课题原始18功能模块部分超出实训可实现范围（AI视觉、GIS、真实支付） | 采用"需求完整定义+编码简化实现"策略，P0/P1完整实现，P2*模拟实现 |
| 需求评审中发现8项遗漏/矛盾/不明确项 | 逐项修订闭合，补充缺失功能点，明确模糊定义 |
| 22模块与3人团队的工作量平衡 | 按优先级分层：P0/P1共9模块（核心必做），P2共3模块（增强），P2*共10模块（简化） |
