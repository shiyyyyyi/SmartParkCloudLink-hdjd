## 1. 产品概述

### 1.1 产品定位

**智慧停车平台** 是一款面向城市停车场景的 Web 管理系统，核心解决 "车主找车位难、车场管理乱、数据不互通" 三大痛点。

### 1.2 产品愿景

> 让每一位车主 "出门就知道停哪儿"，让每一个车场 "数据驱动运营"。

### 1.3 目标用户

| 用户角色 | 核心诉求 | 使用频率 |
|----------|----------|:--------:|
| 私家车主 | 快速查车位、在线预约、无感支付、反向寻车 | 每日 |
| 停车场管理员 | 管理车位状态、查看收入统计、处理异常订单 | 每日 |
| 车位主（共享） | 发布闲置车位时段、获取额外收入 | 按需（V2.0） |
| 运营主管 | 查看全局运营数据、辅助管理决策 | 每周 |

### 1.4 产品范围

| 版本 | 范围 | 状态 |
|------|------|:----:|
| **V1.0** | M1-M9 核心闭环（车主端+管理端） | ✅ 已定义 |
| V2.0 | +M10共享车位 +M11违停管理 +权限细分 | 📋 已规划 |
| V3.0 | UI打磨 + 演示准备 + 答辩 | 📋 已规划 |

## 2. 功能需求

### 2.1 功能全景

```
┌────────── 车主端 (M1-M7) ──────────────┐
│  M1 用户注册/登录                       │
│  M2 停车场信息浏览                      │
│  M3 车位实时查询                        │
│  M4 车位在线预约（并发控制）             │
│  M5 停车计费与支付（免费时段）            │
│  M6 停车记录与反向寻车（纯文字指引）      │
│  M7 车主服务端（统一入口）               │
├────────── 管理端 (M8-M9) ──────────────┤
│  M8 车场运营管理后台                     │
│  M9 数据可视化看板（ECharts×4）          │
├────────── 扩展 (M10-M11) ──────────────┤
│  M10 共享车位管理    M11 违停记录管理    │
└──────────────────────────────────────────┘
```

### 2.2 模块需求明细（关键规则速查）

| 模块 | 优先级 | 核心业务规则 | 负责人 |
|------|:------:|-------------|:------:|
| M1 注册登录 | P0 | 密码≥6位+字母数字；bcrypt哈希存储 | 程晓洋 |
| M2 车场管理 | P0 | 软删除（status标记）；管理员CRUD | 程晓洋 |
| M3 车位查询 | P0 | 实时状态=总量-占用-锁定；按距离排序 | 程子浩 |
| M4 车位预约 | P0 | 15min锁定→超时释放；**同用户仅1个active预约**；行级锁防并发 | 程子浩 |
| M5 计费支付 | P0 | **≤15min免费**；>15min按整时计；订单号UNIQUE幂等 | 程子浩 |
| M6 记录寻车 | P1 | 纯文字指引格式"B2层-西区-128号，直行50米左转" | 丁梓钊 |
| M7 车主首页 | P0 | 搜索框+快捷入口+进行中预约/订单 | 丁梓钊 |
| M8 运营后台 | P1 | V1.0统一admin角色；收入/车流统计 | 程晓洋 |
| M9 数据看板 | P1 | 默认7天/按天聚合；手动刷新；4张ECharts图表 | 程晓洋 |
| M10 共享车位 | P2 | 车位主发布+车主预约（V2.0） | 丁梓钊 |
| M11 违停管理 | P2 | 登记+查询+统计（V2.0） | 丁梓钊 |

### 2.3 车位状态机

```
free → locked(预约) → occupied(到场) → free(出场)
free → locked → free(超时15min/取消)
```

## 3. 非功能性需求

| 类型 | 要求 |
|------|------|
| **性能** | 页面≤2s，API≤500ms |
| **可用性** | Chrome/Edge 90+，移动端响应式 |
| **安全** | bcrypt哈希、Session管理、ORM防注入 |
| **可维护性** | PEP8+ESLint+Git分支管理 |
| **数据规模** | 50+车场×10-30车位=500+，8区域分布 |

## 5. 技术方案

| 层级 | 选型 | 版本 |

### 核心API接口

| 方法 | 路径 | 功能 | 模块 |
|------|------|------|:--:|
| POST | `/api/auth/register` | 用户注册 | M1 |

| GET | `/api/lots` | 车场列表 | M2 |
| POST | `/api/lots` | 新增车场(admin) | M2 |
| GET | `/api/lots/{id}/spots` | 车位列表 | M3 |
| POST | `/api/reservations` | 创建预约(并发校验) | M4 |

| POST | `/api/orders/checkout` | 出场结算(幂等校验) | M5 |
| GET | `/api/orders` | 停车记录 | M6 |
| GET | `/api/orders/current` | 当前停车位置(寻车) | M6 |
| GET | `/api/admin/stats` | 运营统计 | M8 |

## 6. QA质量风险与规避

| 风险 | 规避措施 |
|------|----------|
| 超时状态不一致 | 后端被动检查expire_at |
| 并发预约冲突 | UPDATE行级锁 `WHERE status='free'` |
| 支付重复提交 | 订单号UNIQUE索引 |
| 数据删除追溯 | 软删除（status标记） |
