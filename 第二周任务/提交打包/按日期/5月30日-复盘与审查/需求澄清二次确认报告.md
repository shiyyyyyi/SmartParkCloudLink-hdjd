## 二、二次确认（5月30日）

以产品经理视角，对上述8个点进行二次深度审查：

### 2.1 车位预约的并发控制

```
问题：当多个用户同时预约同一车位时，如何保证不会超卖？

一次结论：乐观锁 + 超时释放（15分钟）
二次确认：
  ├── ✅ 乐观锁适用（车位预约为低频操作，冲突概率低）
  ├── ⚠️ 补充：需增加"预占机制"——用户进入支付页时即锁定车位
  └── ⚠️ 补充：超时后需通知用户"预约已失效"，引导重新预约
    
最终确认：乐观锁（version字段）+ 预占锁定（状态机）+ 15分钟超时释放 + 失效通知
```

### 2.2 支付回调的异常处理

```
问题：支付成功后微信回调失败怎么办？

一次结论：需要幂等性设计
二次确认：
  ├── ✅ 幂等性——订单号唯一索引，重复回调不重复处理
  ├── ⚠️ 补充：主动查询机制——前端定时轮询（每3秒，最多10次）
  └── ⚠️ 补充：对账机制——每日与微信支付账单对账
    
最终确认：幂等回调 + 主动轮询 + 每日对账三重保障
```

### 2.3 室内定位技术选型

```
问题：停车场室内GPS信号弱，如何实现定位？

一次结论：建议蓝牙信标
二次确认：
  ├── ⚠️ 重新评估：蓝牙信标需要硬件部署，超出系统边界
  ├── ✅ 替代方案：基于二维码/车位编号的"拍照定位"——用户停车后拍车位号
  └── ✅ V3.0可演进为蓝牙方案（如果有硬件条件）
    
最终确认：
  V1.0-V2.0：基于车位编号的"标记停车位置"（用户手动记录或扫码）
  V3.0（远期）：蓝牙信标室内定位（需硬件支持）
```

### 2.4 用户隐私与数据安全

```
问题：车牌号、手机号等敏感信息如何保护？

一次结论：需要脱敏+加密
二次确认：
  ├── ✅ 数据库存储：车牌号/手机号 AES-256 加密
  ├── ✅ 前端展示：手机号中间4位脱敏（138****5678）
  ├── ⚠️ 补充：车牌号展示规则——管理者可见全号，普通用户仅看自己的
  └── ⚠️ 补充：需符合《个人信息保护法》要求
    
最终确认：AES-256加密存储 + 前端脱敏展示 + 角色权限控制可见性
```

### 2.5 车牌识别准确率要求

```
问题：车牌识别做不到100%准确怎么办？

一次结论：建议≥95%
二次确认：
  ├── ✅ 95%为行业及格线（主流方案可达98%+）
  ├── ⚠️ 补充：人工纠错入口——识别失败时允许手动输入
  └── ⚠️ 补充：模糊匹配策略——允许1位字符偏差（如0和O易混淆）
    
最终确认：自动识别≥95% + 人工纠错入口 + 模糊匹配容错
```

### 2.6 系统并发用户数预估

```
问题：系统需要支持多少并发？

一次结论：建议1000+ QPS
二次确认：
  ├── ⚠️ 重新评估：课程项目为演示级别，1000+ QPS 过于激进
  ├── ✅ 修正：核心接口（车位查询/支付）≥100 QPS
  ├── ✅ 一般接口（登录/订单查询）≥50 QPS
  └── ✅ 压力测试目标：模拟200并发用户，响应时间<2秒
    
最终确认：
  核心接口≥100 QPS，一般接口≥50 QPS
  压力测试：200并发，P99<2秒
```

### 2.7 管理后台权限模型

```
问题：管理员权限如何划分？

一次结论：RBAC三级权限
二次确认：
  ├── ✅ RBAC模型适用
  ├── ✅ 三级角色：超级管理员 / 运营管理员 / 财务管理员
  └── ⚠️ 补充：需要细化权限矩阵
    
最终确认权限矩阵：

| 功能模块 | 超级管理员 | 运营管理员 | 财务管理员 |
|----------|:--:|:--:|:--:|
| 用户管理 | ✅ R/W | ✅ R | ❌ |
| 车位管理 | ✅ R/W | ✅ R/W | ❌ |
| 订单管理 | ✅ R/W | ✅ R | ✅ R/W |
| 财务管理 | ✅ R/W | ❌ | ✅ R/W |
| 系统配置 | ✅ R/W | ❌ | ❌ |
| 数据报表 | ✅ R/W | ✅ R | ✅ R |
```

### 2.8 数据统计维度与报表需求

```
问题：运营需要哪些数据报表？

一次结论：按日/周/月/车位
二次确认：
  ├── ✅ 时间维度：日报、周报、月报
  ├── ✅ 车位维度：各车位使用率、周转率
  ├── ⚠️ 补充：收入维度——日报收入、支付方式分布
  └── ⚠️ 补充：用户维度——新增用户、活跃用户、复停率
    
最终确认报表需求（V2.0实现）：

| 报表名称 | 维度 | 刷新频率 |
|----------|------|:--:|
| 车位利用率报表 | 车位/时段 | 每小时 |
| 日营收报表 | 日期/支付方式 | 每日 |
| 用户增长报表 | 日期/渠道 | 每日 |
| 运营周报 | 周汇总 | 每周 |
| 月度经营分析 | 月汇总 | 每月 |
```
