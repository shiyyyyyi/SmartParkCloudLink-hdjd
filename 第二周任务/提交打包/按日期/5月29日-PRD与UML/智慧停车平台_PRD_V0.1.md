## 1. 产品概述

### 1.1 产品定位

**智慧停车平台** 是一款面向城市停车场景的 Web 管理系统，核心解决 "车主找车位难、车场管理乱、数据不互通" 三大痛点。产品以车主端（车位搜索→预约→支付→寻车）和管理端（运营后台→数据看板）两条主线，提供完整的停车服务闭环。

### 1.2 产品愿景

> 让每一位车主 "出门就知道停哪儿"，让每一个车场 "数据驱动运营"。

### 1.3 目标用户

| 用户角色 | 核心诉求 | 使用频率 |
|----------|----------|:--------:|
| 私家车主 | 快速查车位、在线预约、无感支付、反向寻车 | 每日 |
| 停车场管理员 | 管理车位状态、查看收入统计、处理异常订单 | 每日 |
| 车位主（共享） | 发布闲置车位时段、获取额外收入 | 按需 |
| 运营主管 | 查看全局运营数据、辅助管理决策 | 每周 |

### 1.4 产品范围

**本期范围（V1.0）**: 车主端 M1-M7 + 管理端 M8-M9 = 9个核心模块

**下期规划（V2.0/V3.0）**: 共享车位 M10 + 违停管理 M11 + 高级分析功能

### 2.2 模块需求明细

#### M1 — 用户注册与登录

| 项 | 说明 |
|----|------|
| **功能描述** | 用户通过手机号+密码注册账号，登录后使用完整功能 |
| **优先级** | P0（核心） |
| **输入** | 手机号、密码、确认密码 |
| **输出** | 注册成功→自动登录；登录成功→跳转车主首页 |
| **业务规则** | ①密码长度≥6位，需包含数字和字母 ②手机号唯一性校验 ③密码哈希存储不存明文 |
| **异常处理** | 手机号已注册→提示"该手机号已注册"；登录失败→提示"用户名或密码错误" |

#### M2 — 停车场信息管理

| 项 | 说明 |
|----|------|
| **功能描述** | 管理员录入/编辑车场信息；车主浏览车场列表与详情 |
| **优先级** | P0（核心） |
| **输入** | 车场名称、地址、经纬度、总车位数、费率（元/小时） |
| **输出** | 车场列表（分页）、车场详情（含实时空余车位数） |
| **业务规则** | ①管理员方可新增/编辑/删除车场 ②车场删除前须无进行中的订单 |
| **异常处理** | 数据校验失败→返回具体字段错误提示 |

#### M3 — 车位状态实时查询

| 项 | 说明 |
|----|------|
| **功能描述** | 车主按目的地搜索附近车场，查看各车场实时空余车位 |
| **优先级** | P0（核心） |
| **输入** | 搜索关键词（目的地/地址）、筛选条件（距离/价格/空位数） |
| **输出** | 车场列表（含名称、地址、距离、空余车位、费率） |
| **业务规则** | ①默认按距离排序 ②支持按价格/空位数排序 ③空余车位=总车位-已占用-已锁定 |
| **异常处理** | 无匹配结果→提示"未找到相关停车场" |

#### M4 — 车位在线预约

| 项 | 说明 |
|----|------|
| **功能描述** | 车主选定车位后预约锁定15分钟，到期自动释放 |
| **优先级** | P0（核心） |
| **输入** | 用户ID、车位ID、车场ID |
| **输出** | 预约成功→显示倒计时（15分钟）→到场确认/取消预约 |
| **业务规则** | ①车位状态机：空闲→锁定→占用→空闲 ②锁定15分钟超时自动释放 ③同一用户同时只能预约1个车位 |
| **异常处理** | 车位已被锁定→提示"该车位已被预约"；超时未确认→自动释放 |

#### M5 — 停车计费与支付

| 项 | 说明 |
|----|------|
| **功能描述** | 记录入场/出场时间，按费率计算费用，模拟支付完成结算 |
| **优先级** | P0（核心） |
| **输入** | 入场时间（系统自动记录）、出场时间、费率 |
| **输出** | 费用明细（时长、单价、总价）→确认支付→生成订单 |
| **业务规则** | ①计费=时长(小时)×费率(元/小时)，不足1小时按1小时计 ②模拟支付：前端模拟支付流程，后端直接变更状态 ③订单状态：进行中→已完成 |
| **异常处理** | 支付确认失败→保留订单，可重新发起 |

#### M6 — 停车记录与反向寻车

| 项 | 说明 |
|----|------|
| **功能描述** | 车主查看历史停车记录；输入车牌号查看当前停车位置和步行指引 |
| **优先级** | P1（重要） |
| **输入** | 车牌号（查当前）、筛选条件（查历史） |
| **输出** | 历史记录列表；当前车位编号+区域描述+简单步行指引 |
| **业务规则** | ①仅展示本人停车记录 ②反向寻车：输入车牌号→匹配当前占用车位→显示位置信息 |
| **异常处理** | 未找到→提示"未找到该车辆当前停车记录" |

#### M7 — 车主服务端（统一入口）

| 项 | 说明 |
|----|------|
| **功能描述** | 车主首页，整合搜索车位、我的预约、停车记录、快捷入口 |
| **优先级** | P0（核心） |
| **输入** | 无（首页加载） |
| **输出** | 搜索框、快捷入口（我的预约/停车记录/找我的车）、推荐车场 |
| **业务规则** | ①未登录跳转登录页 ②已登录展示个性化信息（用户名+进行中的预约/订单） |

#### M8 — 车场运营管理后台

| 项 | 说明 |
|----|------|
| **功能描述** | 管理员查看所管车场的车位状态、收入统计、进出流量 |
| **优先级** | P1（重要） |
| **输入** | 管理员登录、车场选择 |
| **输出** | 车位占用一览、当日/本周/本月收入、车流统计 |
| **业务规则** | ①管理员仅能查看所管辖的车场 ②收入数据实时计算 |
| **异常处理** | 无管辖车场→提示"暂无管理权限" |

#### M9 — 数据可视化看板

| 项 | 说明 |
|----|------|
| **功能描述** | ECharts可视化展示车位周转率、高峰饱和度、收入趋势等多维度运营指标 |
| **优先级** | P1（重要） |
| **输入** | 车场ID、时间范围（默认最近7天） |
| **输出** | ①车位周转率折线图 ②高峰时段柱状图 ③收入趋势图 ④车场利用率饼图 |
| **业务规则** | ①车位周转率=总停车次数/总车位数 ②高峰饱和度=最高占用车位数/总车位数 |
| **异常处理** | 数据为空→图表显示空状态提示 |

#### M10 — 共享车位管理（V2.0规划）

| 项 | 说明 |
|----|------|
| **功能描述** | 车位主发布闲置时段出租，车主预约共享车位 |
| **优先级** | P2（增强） |
| **简要说明** | ①车位主设置可共享时段和价格 ②车主搜索共享车位并预约 ③到期自动释放 |

#### M11 — 违停记录管理（V2.0规划）

| 项 | 说明 |
|----|------|
| **功能描述** | 管理员记录违停信息，支持查询和统计 |
| **优先级** | P2（增强） |
| **简要说明** | ①车牌号+违停描述+拍照记录 ②违停列表+筛选查询 ③违停统计 |

## 4. 数据需求

### 4.1 核心数据表（6张）

```sql
-- 1. 用户表
users (id, username, phone, password_hash, role, created_at)

-- 2. 停车场表
parking_lots (id, name, address, latitude, longitude, total_spots, rate_per_hour, admin_id)

-- 3. 车位表
parking_spots (id, lot_id, spot_number, floor, zone, status, updated_at)
-- status: free | locked | occupied | disabled

-- 4. 预约记录表
reservations (id, user_id, spot_id, lot_id, status, created_at, expire_at, confirm_at)
-- status: active | confirmed | cancelled | expired

-- 5. 停车订单表
parking_orders (id, user_id, spot_id, lot_id, entry_time, exit_time, duration_minutes, amount, status)
-- status: parking | completed | cancelled

-- 6. 违停记录表
violations (id, lot_id, spot_id, plate_number, description, created_at, status)
```

### 4.2 车位状态机

```
         ┌─────────┐
         │  free   │ ← 初始状态
         └────┬────┘
              │ 预约 → lock 15分钟
         ┌────▼────┐
         │ locked  │
         └────┬────┘
          ╱       ╲
   到场确认    超时/取消
    ╱             ╲
┌──▼──────┐   ┌───▼───┐
│occupied │   │ free  │ (回到空闲)
└────┬────┘   └───────┘
     │ 出场结算
┌────▼────┐
│  free   │ (释放)
└─────────┘
```

## 5. 技术方案

| 层级 | 技术选型 | 版本 | 说明 |
|------|----------|------|------|
| 前端 | Vue 3 + Element Plus + ECharts | 3.x | 组件化UI、数据可视化 |
| 后端 | FastAPI (Python) | 0.100+ | 异步高性能、Swagger自动文档 |
| 数据库 | SQLite + SQLAlchemy | 内置 | 零部署、轻量级 |
| 支付 | 模拟支付 | — | 前端模拟流程，不接入真实支付 |
| 地图 | 静态示意图 | — | 停车场列表+简单区域示意 |

| 路由 | 页面 | 权限 |
