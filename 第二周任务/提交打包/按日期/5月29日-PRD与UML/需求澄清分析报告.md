# 需求澄清分析报告（Requirements Clarity）

> **PRD版本**: V0.1 | **分析日期**: 2026-05-29 | **分析师**: 程子浩 (SA/QA)
>
> 本报告基于 requirements-clarity 技能对 PRD V0.1 进行系统性模糊点扫描，共识别 **8个模糊点**，按严重程度分为高/中/低三档。

---

## 一、已识别模糊点

### 模糊点 #1 — 预约超时后的车位释放是否通知用户？【严重程度：中】

| 项 | 说明 |
|----|------|
| **模糊描述** | PRD中提到"15分钟超时自动释放"，但未明确：释放后是否主动通知车主？以什么方式通知？ |
| **当前状态** | 仅描述了系统自动释放，无通知逻辑 |
| **影响范围** | M4 车位在线预约 |
| **澄清建议** | ①前端页面倒计时归零后弹窗提示"预约已过期" ②同时将预约记录状态更新为expired，车主在"我的预约"中可看到过期记录 |
| **PM回复** | _待回复_ |

---

### 模糊点 #2 — 计费规则"不足1小时按1小时计"是否包含免费时段？【严重程度：高】

| 项 | 说明 |
|----|------|
| **模糊描述** | PRD中计费规则为"不足1小时按1小时计"，但未考虑：①是否有免费时段（如进场15分钟内免费）②跨天/跨费率时段如何计费 |
| **当前状态** | 统一按小时计，无梯度逻辑 |
| **影响范围** | M5 停车计费与支付 |
| **澄清建议** | 明确规则：①前15分钟免费（不足15分钟不计费）②超过15分钟按1小时起计 ③暂不考虑跨时段费率差异（V1.0简化） |
| **PM回复** | _待回复_ |

---

### 模糊点 #3 — 反向寻车的"步行指引"具体以什么形式呈现？【严重程度：中】

| 项 | 说明 |
|----|------|
| **模糊描述** | PRD描述为"文字+简单示意图"，但未说明示意图是静态图还是交互式地图 |
| **当前状态** | 描述模糊，开发者无法明确实现方式 |
| **影响范围** | M6 反向寻车 |
| **澄清建议** | V1.0采用纯文字+静态区域平面图（手绘/简单标注），不做交互式地图。展示格式：显示车场区域划分图 + 标注当前位置和车位位置 + 文字步行路线 |
| **PM回复** | _待回复_ |

---

### 模糊点 #4 — 管理员权限粒度是否区分超级管理员和普通管理员？【严重程度：中】

| 项 | 说明 |
|----|------|
| **模糊描述** | PRD中管理员角色未细分：是统一管理员还是分级管理？普通管理员能否管理所有车场？ |
| **当前状态** | 仅一个admin角色，无权限细分 |
| **影响范围** | M8 运营管理后台 |
| **澄清建议** | V1.0简化为统一管理员角色（admin），不做权限细分。admin可管理所有车场数据 |
| **PM回复** | _待回复_ |

---

### 模糊点 #5 — 数据看板的统计时间粒度没有明确【严重程度：低】

| 项 | 说明 |
|----|------|
| **模糊描述** | PRD提到"最近7天数据"，但按小时/按天聚合？实时更新还是定时刷新？ |
| **当前状态** | 仅提到默认7天，未定义粒度 |
| **影响范围** | M9 数据可视化看板 |
| **澄清建议** | ①默认展示最近7天数据，按天聚合 ②支持切换时间范围（今日/本周/本月）③页面不自动刷新，需手动刷新 |
| **PM回复** | _待回复_ |

---

### 模糊点 #6 — 车主能否同时拥有多个进行中的预约？【严重程度：高】

| 项 | 说明 |
|----|------|
| **模糊描述** | PRD中提到"同一用户同时只能预约1个车位"，但未明确：用户是否可以在不同车场分别预约？已预约状态是否阻止再次预约？ |
| **当前状态** | 规则描述不够严密 |
| **影响范围** | M4 车位在线预约 |
| **澄清建议** | 规则统一为：①同一用户在任何时刻只能持有1个active预约 ②预约未确认/未超时前，前端禁止发起新预约 ③后端也需校验：创建预约前检查用户是否有进行中的active预约 |
| **PM回复** | _待回复_ |

---

### 模糊点 #7 — 数据库表结构的字段类型和约束未定义【严重程度：中】

| 项 | 说明 |
|----|------|
| **模糊描述** | PRD中数据表仅给出字段名，未定义数据类型、长度、是否可空、默认值、索引等 |
| **当前状态** | 仅SQL伪代码，无法直接建表 |
| **影响范围** | 全局（所有模块依赖） |
| **澄清建议** | 需要DBA（程晓洋）补充完整DDL：字段类型（VARCHAR长度/INT/DATETIME）、NOT NULL约束、默认值、外键关系、索引定义 |
| **PM回复** | _待回复_ |

---

### 模糊点 #8 — 测试数据准备的标准和范围是什么？【严重程度：低】

| 项 | 说明 |
|----|------|
| **模糊描述** | 提到"50+停车场、500+车位模拟数据"，但未明确数据分布要求（区域分布、价格区间、车位类型比例等） |
| **当前状态** | 仅有数量指标 |
| **影响范围** | 全局（测试阶段） |
| **澄清建议** | 测试数据要求：①5-8个区域各分配若干车场 ②价格范围3-15元/小时 ③每个车场10-30个车位 ④车位初始状态90%空闲、5%占用、5%锁定 ⑤3-5个管理员账号 |
| **PM回复** | _待回复_ |
