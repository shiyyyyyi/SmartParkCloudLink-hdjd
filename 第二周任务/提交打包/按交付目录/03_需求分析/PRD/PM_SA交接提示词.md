## 三、PRD V0.2 核心要点速览

### 功能范围

```
V1.0（9模块 ─ 核心闭环）:
  M1-M7 车主端：注册→搜索→查询→预约→支付→寻车→首页
  M8-M9 管理端：运营后台 + 数据看板

V2.0（11模块 ─ 功能增强）:
  + M10 共享车位 + M11 违停管理 + 权限细分

V3.0（打磨）:
  UI优化 + 演示准备 + 答辩
```

### 关键业务规则（SA必须确保开发团队理解）

| # | 规则 | 涉及模块 | 优先级 |
|:--:|------|:--------:|:------:|
| 1 | 前15分钟免费，超过按整小时计 | M5 | 🔴 高 |
| 2 | 同一用户只能持有1个active预约（前后端双重校验） | M4 | 🔴 高 |
| 3 | 预约15分钟超时自动释放+前端通知 | M4 | 🟡 中 |
| 4 | 车位并发预约使用行级锁 | M4 | 🔴 高 |
| 5 | 支付幂等性（订单号UNIQUE） | M5 | 🔴 高 |
| 6 | 车场删除采用软删除 | M2 | 🟡 中 |
| 7 | 反向寻车V1.0纯文字指引 | M6 | 🟢 低 |
| 8 | 管理员V1.0统一角色不细分 | M8 | 🟡 中 |

### 车位状态机（三个开发者必须对齐）

```
free → locked (预约) → occupied (到场确认) → free (出场结算)
free → locked → free (超时15分钟/取消)
```
